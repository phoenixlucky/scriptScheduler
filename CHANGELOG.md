# Changelog

## 1.2.2 - 2026-04-22

- 修复 `tasks.json` 损坏或格式非法时应用启动失败的问题；现在会自动备份损坏文件并重建空任务库继续启动。
- 补充 README 中的旧打包产物清理命令和 Windows 安装包打包说明。

## 1.2.1 - 2026-03-25

- 新增最小化到系统托盘能力，关闭窗口后调度器仍可在后台继续运行。
- 补充软件中英文简介、核心定位和版本说明，统一更新关于页元数据。
- 安装器按系统语言显示名称：英文环境为 `WeiScheduler`，中文环境为 `尉定时任务调度器`。

## 1.2.0 - 2026-03-25

- 修复跨小时 Cron 任务的稳定性问题，为 `每 3 小时`、`每 6 小时` 等表达式增加服务端补偿触发，避免整点漏执行。
- 新增 `每 6 小时` 预设，并在任务列表中显示对应的人类可读描述，方便核对任务频率。

## 1.1.10 - 2026-03-24

- 任务列表新增“下次执行”具体时间显示，直接展示下一次 Cron 触发时刻。
- 任务详情新增“下次执行”字段，方便核对任务是否按预期调度。

## 1.1.9 - 2026-03-23

- 优化 Conda 环境解析：优先读取 `conda env list --json` / `conda info --envs --json`，按环境名定位真实路径，减少对固定盘符和用户名的依赖。
- 为 `conda-name` / `conda-path` 增加执行回退策略：找不到 `python.exe` 时自动回退到 `conda run --no-capture-output`。
- 改进手动执行失败提示：点击“立即执行”或“立即执行1次”失败时，弹窗显示更具体的错误内容。
- 改进任务状态展示：失败任务在列表中显示更明确的失败原因。
- 新增 Cron 五段自由组合输入，并与原始 Cron 表达式、预设按钮双向同步。
- 修复失败日志可能被覆盖、导致只看到 `failed` 看不到具体错误的问题。

## 1.1.5

- 优化关于对话框和应用元数据。

## 1.1.1

- 发布 Windows 安装包。
